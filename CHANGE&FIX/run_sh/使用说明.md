# run.sh 使用说明（中文）

`run.sh` 是对 `nextflow run` 的封装脚本，用于统一管理 Riboseq 流程的启动/停止、诊断检查、日志与结果查看。

## 1) 快速开始

```bash
chmod +x run.sh
./run.sh diagnose
./run.sh start
./run.sh logs-follow
```

## 2) 运行前提

- Nextflow 可在当前 shell 中直接调用（建议：`conda activate nextflow`）
- 使用 `RUNWAY=docker` 时需确保 Docker 正常运行

## 3) 常用命令

以 `./run.sh help` 为准，最常用的是：

```bash
./run.sh diagnose
./run.sh start|stop|restart|status
./run.sh logs|logs-follow|logs-error|logs-search <关键词>
./run.sh summary|report|results
./run.sh clean
```

## 4) 配置方式（环境变量）

`run.sh` 通过环境变量读取配置，默认值如下：

```bash
PIPELINE_DIR=./                    # 默认：./（传给 nextflow run）
CONFIG_FILE=conf/test_local.config  # 默认：conf/test_local.config（传给 -c）
WORK_DIR=nextflow_work             # 默认：nextflow_work（传给 -w）
RUNWAY=docker                      # 默认：docker（传给 -profile）
OUTDIR=results_testdata            # 默认：results_testdata（传给 --outdir）
EXTRA_ARGS=""                      # 默认：空（追加到 nextflow run）
```

示例（按需覆盖）：

```bash
PIPELINE_DIR="/path/to/pipeline" \
CONFIG_FILE="conf/my.config" \
RUNWAY="docker" \
OUTDIR="results" \
EXTRA_ARGS="--max_cpus 8" \
./run.sh start
```

## 5) 产物位置（最常用）

- Nextflow 日志：`.nextflow.log`
- 启动输出：`nextflow.out`
- 工作目录：`nextflow_work/`（或 `WORK_DIR`）
- 结果目录：`results_testdata/`（或 `OUTDIR`）

## 6) 典型工作流

首次运行：

```bash
./run.sh diagnose
./run.sh start
./run.sh logs-follow
```

日常监控：

```bash
./run.sh status
./run.sh logs-error
```

完成后：

```bash
./run.sh summary
./run.sh report
./run.sh clean
```

## 7) 快速排错

```bash
./run.sh diagnose
./run.sh check-config
./run.sh check-docker
./run.sh logs-error
```

若流程未启动或立即退出，优先查看：

```bash
tail -n 200 nextflow.out
tail -n 200 .nextflow.log
```
